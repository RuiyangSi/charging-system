# 验收用例填写结果

本目录存放依据本系统实际运行结果填写的《作业验收用例》。

## 文件
- `作业验收用例_本组填写.xlsx` —— 由 `backend/fill_acceptance.py` 跑完整 42 个验收事件自动填写：
  - **测试用例**：每个时刻 5 个充电桩的 `(车号, 已充电量, 当前费用)` + 等候区 `(车号, 模式, 请求电量)`。
  - **账单结果**：逐张账单（车号 / 计费量(度) / 费用(元) / 说明）+ 总营收。
  - **测试说明**：沿用老师模板原页。

## 复现方式
```bash
cd backend
python3 fill_acceptance.py        # 读取 ../../作业验收用例.xlsx 模板，输出到 ../docs/
```

## 关键结果与验证
- **调度结束 = 次日 01:35**，与老师模板预置的 `1:35+1` 一致（`python3 replay_acceptance.py` 可复核）。
- 老师给定样例对齐：`06:05` 慢充1 = `(V1,0.83,1.00)`、慢充2 = `(V2,0.00,0.00)`。
- 账单合计 28 笔、总营收 ¥2025.5；所有车辆最终均结清（无滞留、无悬空中断）。
- 单元测试 `python3 -m pytest tests/` 全绿。

## 故障再调度口径（与结果一致）
系统采用 `requeue`（config 默认）：充电桩故障时，正在充电的车按**已充电量先部分计费**，其**剩余电量重新入队续充**；排队车按排队号重排。因此被中断的车（如 V1、V3）会先出一张「故障中断」账单，剩余电量充完后再出一张账单。该口径满足老师的优先级调度规程，且最终调度结束时刻为 01:35。
